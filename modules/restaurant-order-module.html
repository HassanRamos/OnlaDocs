Restaurant Order Module

Monitor customer orders and kitchen tickets, including KOT status such as preparing, ready, cancelled, or completed depending on your setup.

What this module is forMain featuresTypical workflowGood practiceScreens / Visual Guide

What this module is for

  • The order module keeps track of customer requests for food and drinks.
  • It also helps the team monitor kitchen tickets so staff know which orders are pending, ready, or cancelled.
  • This module is useful for operations control after an order has been placed.

Main features

  • View KOT records.
  • Change KOT status when food is ready or when a ticket must be cancelled.
  • View order records and related order information.
  • Track order progress from creation to completion.

Typical workflow

  1. Open the Orders or KOT section.
  2. Review pending kitchen tickets.
  3. Mark food as ready when preparation is complete.
  4. Cancel a KOT only when the order should not be prepared.
  5. Open the order list to confirm billing/payment status.

Good practice

  • Use clear order statuses so waiters and kitchen staff stay aligned.
  • Do not cancel KOTs without confirming with the responsible staff member.
  • Use the order list to resolve customer disputes about what was ordered or paid.
  • Review open orders before end-of-day closing.
