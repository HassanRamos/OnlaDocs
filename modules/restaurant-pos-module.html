Restaurant POS Module

Use the Point of Sale screen to create orders, send KOTs, process bills, and collect payments during daily restaurant service.

What this module is forMain featuresTypical workflowGood practiceScreens / Visual Guide

What this module is for

  • The POS screen is used by cashiers, waiters, or managers to place customer orders and process sales.
  • It connects front-of-house ordering with kitchen preparation and payment collection.
  • KOT means Kitchen Order Ticket. It tells the kitchen what items need to be prepared.

Main features

  • Select tables or order types depending on your setup.
  • Add menu items to an order.
  • Send items to the kitchen as KOT.
  • Update quantities, notes, and order details.
  • Generate bills and collect payments.
  • Handle dine-in, takeaway, or delivery-style orders where supported.

Typical workflow

  1. Open the POS module.
  2. Select the table or customer/order type.
  3. Add menu items to the cart.
  4. Send KOT to the kitchen if preparation is required.
  5. When the customer is ready, generate the bill.
  6. Collect payment and close the order.

Good practice

  • Confirm table number before sending KOT.
  • Use order notes for kitchen instructions such as “no sugar”, “extra spicy”, or “no onions”.
  • Ensure payments are recorded before closing orders.
  • Train cashiers and waiters on the difference between saving, sending KOT, billing, and closing an order.
