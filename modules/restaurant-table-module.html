Restaurant Table Module

Manage restaurant areas, dining tables, seating capacity, table status, and QR codes for table-based ordering.

What this module is forMain featuresTypical workflowGood practiceScreens / Visual Guide

What this module is for

  • Use the table module to set up the physical layout of the restaurant.
  • Areas can represent sections such as lounge, garden, terrace, VIP area, bar area, or main dining area.
  • Tables are created under these areas and can be linked to orders, KOTs, and QR ordering.

Main features

  • Create multiple dining areas.
  • Add tables with table code/table number and seating capacity.
  • View table status such as available, running, occupied, or order-related activity depending on your setup.
  • Access table-related actions such as orders, KOT, edit, and availability.
  • Generate or view QR codes linked to individual tables.

Typical workflow

  1. Open the Table or Area section.
  2. Create the restaurant areas first.
  3. Add tables under the correct area.
  4. Enter the table number/code and seating capacity.
  5. Use QR code options when customers will scan table codes for self-ordering.

Good practice

  • Name areas the same way staff use them inside the restaurant.
  • Keep table codes short and easy to identify.
  • Update table availability when the restaurant layout changes.
  • Print and test QR codes before placing them physically on tables.
