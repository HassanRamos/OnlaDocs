Menu-to-Table Assignment

Control which menu items appear for specific tables when customers order through table QR codes.

What this module is forWhere it helpsMain featuresTypical workflowImportant notesScreens / Visual Guide

What this module is for

  • Menu-to-table assignment controls menu visibility by table.
  • It is mainly useful for QR-code ordering because the system can detect the table from the scanned QR code.
  • You can show different items to different areas or table groups without deleting items globally.

Where it helps

  • Bar-only sections.
  • VIP lounge items.
  • Outdoor seating limitations.
  • Breakfast/lunch/dinner table zones.
  • Special event tables.
  • Kitchen availability rules.

Main features

  • Assign selected menu items to a specific table.
  • Assign items to multiple tables where supported.
  • Allow a table to have multiple visible items/menus.
  • Hide items from tables where they should not be available.
  • Automatically filter customer-facing menus when the customer scans a table QR code.

Typical workflow

  1. Open the Menu Management page.
  2. Choose the Assign Menu to Table option.
  3. Select the table.
  4. Select the menu items that should be visible for that table.
  5. Save the assignment.
  6. Test using that table’s QR code and confirm only the correct items appear.

Important notes

  • This feature depends on table identification, so it works best through table-specific QR codes.
  • If a customer opens the menu using a general link that does not identify a table, table-based filtering may not apply.
  • Always test table QR codes after making changes.
